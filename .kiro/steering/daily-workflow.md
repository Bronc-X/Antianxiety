# Daily Workflow Rules

> 每日工作流规则 - 确保文档及时更新和营销素材收集

---

## 📅 每日开工检查清单

在开始每日开发工作前，请完成以下检查：

### 1. 阅读项目宪法
- [ ] 查看 `PROJECT_CONSTITUTION.md` 确认核心理念和编码规范
- [ ] 特别关注 Section I (Philosophy & Vibe) 和 Section IV (Coding Standards)

### 2. 检查最近开发日志
- [ ] 阅读 `DEVELOPMENT_DIARY.md` 最近 2-3 天的记录
- [ ] 了解当前进展和待办事项
- [ ] 确认是否有未完成的任务需要继续

### 3. 查看待处理营销素材
- [ ] 检查 `MARKETING_ASSETS.md` 中的 TODO 条目
- [ ] 特别关注逾期（超过3天）的素材
- [ ] 如有紧急素材（超过7天），优先处理

---

## 🌙 每日结束检查清单

在结束每日开发工作前，请完成以下任务：

### 1. 更新开发日志
- [ ] 在 `DEVELOPMENT_DIARY.md` 添加今日记录
- [ ] **⚠️ 倒序排列**：新日期放在文件最前面（12.2 → 12.1 → 11.30...）
- [ ] 使用标准模板格式（见下方）
- [ ] 包含：核心更新、代码统计、下一步计划

### 2. 检查 README 更新需求
- [ ] 如果完成了重大功能，更新 `README.md`
- [ ] 重大功能定义：用户可见的新功能、架构变更、新增页面

### 3. 记录营销素材
- [ ] 如果实现了新 UI 组件/页面，添加截图 TODO
- [ ] 如果实现了动画/交互效果，添加录屏 TODO
- [ ] 更新 `MARKETING_ASSETS.md`

---

## 📝 开发日志模板

```markdown
## YYYY-MM-DD - 简短标题

### 🎯 核心更新

#### 1. 功能名称
- ✅ 完成项 1
- ✅ 完成项 2

### 📊 代码统计
- **文件变更**: X 个文件
- **新增代码**: X 行
- **删除代码**: X 行

### 🚀 下一步计划
- [ ] 待办 1
- [ ] 待办 2

### 📸 营销素材
- [ ] 截图: 功能名称
- [ ] 录屏: 动画效果
```

---

## 🔄 架构/工作流变更时

当检测到以下变更时，必须立即更新 `TECH_STACK_AND_WORKFLOW.md`：

- 新增依赖包
- 修改构建流程
- 新增 API 端点
- 修改数据库 Schema
- 新增 Kiro Hooks 或 Steering 规则

---

## ⚠️ 重要提醒

1. **不要拖延文档更新** - 当天的工作当天记录
2. **不要遗漏营销素材** - 功能完成后立即截图/录屏
3. **保持格式一致** - 使用标准模板
4. **定期回顾** - 每周检查一次逾期素材
5. **倒序排列日志** - 新的日期永远在最前面，方便快速查看最新进展
