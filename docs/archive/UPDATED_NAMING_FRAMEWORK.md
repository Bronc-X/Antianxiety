# 去AI化训练周期命名框架

## 设计原则修订

### 1. **去除AI味道**
- 避免：AI、智能、算法、精准、数据驱动等科技术语
- 采用：训练、调理、养成、恢复、提升等体感词汇

### 2. **内容关联性**
- 根据具体方案内容命名
- 体现训练的核心目标和方法
- 突出用户能感知到的变化

### 3. **训练周期融入**
- 明确标注训练周期：7天、14天、21天、30天
- 使用阶段性概念：基础期、强化期、稳定期
- 体现渐进式训练理念

## 基于现有内容的具体命名

### 当前方案内容分析：
1. **延迟早餐至10:00** → 间歇性禁食训练
2. **睡前镁离子补充** → 深度睡眠优化  
3. **HIIT转Zone 2** → 有氧基础重建

### 新命名建议：

#### 主方案：
```
🔄 7天代谢重置训练营
```
**理由**：
- 去除"高效能人士"等标签化表达
- "代谢重置"直接关联方案内容
- "训练营"体现系统性和专业性
- "7天"明确训练周期

#### 具体策略重命名：
```
📅 生物钟调节训练：延迟进食至10:00（间歇性禁食）
🌙 深度睡眠训练：睡前1小时镁元素补充
🏃 有氧基础重建：暂停高强度，专注Zone 2训练
```

## 完整方案命名结构

### 格式：[训练类型] + [周期] + [核心目标] + 训练营/计划/周期

**示例应用：**
```
当前内容 → 新命名

"高效能人士7天精力复活" 
↓
"🔄 7天代谢重置训练营"

"精英专属抗衰分子库"
↓  
"🌿 21天逆龄食材训练手册"

"优化生物钟：延迟早餐"
↓
"📅 生物钟调节训练：间歇性禁食法"
```

## 训练周期设定逻辑

### 7天周期：快速重置类
- 适用：精力恢复、代谢激活、睡眠调整
- 用户期望：立竿见影的改善

### 14天周期：习惯养成类  
- 适用：运动计划、饮食调整、作息规律
- 用户期望：形成新的行为模式

### 21天周期：深度调理类
- 适用：体质改善、压力管理、综合提升
- 用户期望：根本性的身体变化

### 30天周期：系统重建类
- 适用：全面改造、长期目标、生活方式
- 用户期望：持久的健康状态

这个新框架完全去除了AI和科技感，更注重实际的训练内容和周期性，让用户感受到的是专业的健康训练指导，而不是冷冰冰的技术产品。
